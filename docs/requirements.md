# 需求与功能清单

目标：做一个类似「读通鉴」(www.dutongjian.com) 的古籍深度阅读 Web 应用，但采用**通用模板**架构——导入任何符合数据包规范的古籍即可获得完整阅读体验。第一本书为《三国志》，后续计划扩展（史记等）。

## 「读通鉴」功能对照表

| 功能 | 读通鉴的形态 | 本项目状态 |
|---|---|---|
| 目录树导航 | 卷→帝纪→年份，左侧树 | ✅ M1（任意深度树，数据驱动） |
| 文言/白话左右对照 | 逐段对齐，双栏 | ✅ M1 |
| 悬停联动 | 悬停原文高亮对应白话 | ✅ M1（段级） |
| 行内注释（胡注/裴注） | 点击〔注〕展开 | ✅ M1（注释层可配置） |
| 人名生卒年/地名标注 | 人名下标生卒、地名加框 | ✅ M1（实体词典自动标注） |
| 阅读设置 | 简繁/白话显隐/字体/字号/生卒开关 | ✅ M1 |
| 段落收藏（星标） | 登录后云端 | ✅ M1（本地 localStorage） |
| 阅读进度记忆 | 有 | ✅ M1（每书最后章节） |
| 响应式/移动端 | 有 | ✅ M1（窄屏上下堆叠） |

## 非功能需求

- **通用性**：新书零代码接入——提供数据包（book.json/toc.json/chapters）即上架
- **纯前端**：M1 阶段无后端，静态托管即可部署；用户态存 localStorage
- **数据可再生**：`sources/`（原文+翻译+词典）是唯一事实来源，`public/data/` 由脚本生成
- **质量门槛**：main 分支 `npm test`（数据校验+27 测试）必须通过

## 样例内容范围（M1）

《三国志》：魏书·武帝纪（前 12 段，裴注密集）+ 蜀书·诸葛亮传（全文 40 段，含出师表、后出师表〔裴注〕、诸葛氏集目录、陈寿上表、评曰）。原文取自维基文库（公有领域），白话翻译为本项目自制（Claude 生成，人工抽查）。
